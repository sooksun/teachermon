- **packages/database/prisma/seed-indicators.ts** – WP_2 = "การจัดการเรียนรู้ที่เน้นผู้เรียนเป็นสำคัญ" มีคำอธิบายรวม "การจัดกิจกรรมการเรียนรู้แบบ Active Learning"

## 2. การปรับปรุงที่ดำเนินการ

### 2.1 ฐานข้อมูล (Schema)

- **DevelopmentPlan**
  - เพิ่ม `budgetAllocated` (Decimal) – งบประมาณที่จัดสรร (บาท)
  - เพิ่ม `budgetUsed` (Decimal) – งบที่ใช้ไป (บาท)
  - (ถ้าต้องการละเอียดต่อรายการ อาจเพิ่มตาราง PlanExpense ในอนาคต)

### 2.2 API

- **Assessment (Plans)**: รองรับการอ่าน/เขียน `budgetAllocated`, `budgetUsed`
- **Dashboard**: เพิ่มสถิติ
  - สรุปความคืบหน้าแผน (จำนวนแผนตามสถานะ: DRAFT, IN_PROGRESS, COMPLETED, CANCELLED)
  - สรุปงบประมาณ (รวมจัดสรร, รวมใช้แล้ว, คงเหลือ)
  - (ถ้ามี) จำนวนครูที่โฟกัสพัฒนาด้าน Active Learning (WP.2)

### 2.3 ฟรอนต์เอนด์

- **แผนพัฒนารายบุคคล (IDP)**
  - เพิ่มช่องกรอกงบประมาณ (จัดสรร / ใช้ไป) ในฟอร์มสร้าง/แก้ไขแผน
  - แสดงสถานะความคืบหน้าและงบในรายการแผน
- **Dashboard**
  - การ์ด/กราฟความคืบหน้าแผน (ตามสถานะ)
  - การ์ดงบประมาณ (รวมจัดสรร, ใช้แล้ว, คงเหลือ)
- **Active Learning**
  - ใส่ป้าย/ข้อความว่า "Active Learning" สำหรับสมรรถนะ WP.2 ใน dropdown และรายการแผน
  - ตัวกรองแผนหรือครูตามสมรรถนะ (เช่น เลือกเฉพาะ WP.2)

## 3. ตัวชี้วัดที่เกี่ยวข้องกับ Active Learning (อ้างอิง doc_ref3 / INDICATOR)

- **WP_1.3** – การจัดกิจกรรมการเรียนรู้  
  - "มีการจัดการเรียนรู้โดยใช้รูปแบบ เทคนิค และเน้นกระบวนการ **Active Learning**"
- **WP_2** – การจัดการเรียนรู้ที่เน้นผู้เรียนเป็นสำคัญ  
  - รวมการจัดกิจกรรมการเรียนรู้แบบ Active Learning การใช้สื่อและเทคโนโลยี

ดังนั้นการ "ติดตามและพัฒนาครูด้าน Active Learning" จึงสอดคล้องกับการติดตามและส่งเสริมตัวชี้วัด **WP_2** และ **WP_1.3** ในระบบ

## 4. ขั้นตอนถัดไป (แนะนำ)

1. รัน migration หลังเพิ่มฟิลด์งบประมาณใน `development_plan`
2. อัปเดต API (assessment service/controller) และ DTO ให้รับ/ส่ง budgetAllocated, budgetUsed
3. อัปเดต Dashboard service ให้รวมสถิติแผนและงบประมาณ
4. อัปเดตฟอร์มและหน้า Dashboard ตามรายการใน 2.3
5. (ถ้ามีเนื้อหาเพิ่มจาก doc_ref5.pdf) นำข้อความหรือโครงจาก PDF มาใส่ในเอกสารนี้หรือใน PRD เพื่ออ้างอิงในรอบถัดไป
